Julia Set plays with the idea of taking the result computed from a function and recycling it back into the input. Consider the equation y = x2 + c for some constant called c. But now y becomes the next value of x and the next value in a sequence of numbers described by the equation xn+1 = xn2 + c. If we let x0 equal 2 and c equal 0 then we get the sequence 2, 4, 16, 256, 65536… Each number is the square of one before it and the numbers grow larger and larger. But if x0 is ½ then we get the sequence ½, ¼, ¹⁄₁₆, ¹⁄₂₅₆, ¹⁄₆₅₅₃₆… Now the numbers shrink down towards zero. And if x0 is 1 then we get the sequence 1, 1, 1, 1…

Do you know what the square root of -1 is? If we stick to using the ordinary real numbers then there isn't one. A positive number multiplied by itself will always be equal to another positive number and multiplying a negative number by itself will also result in a positive number. There is no real number that when you square it produces a negative number and therefore the square root of -1 has no solution. But that's okay because a mathematician is happy to simply imagine that a solution exists and give it the name i. The imaginary numbers are multiples of this imaginary solution to the square root of minus one. For example 3i is three times the imaginary solution to square root of minus one. A complex number is the sum of a real number and an imaginary number, for example 2 + 3i. This sounds pretty silly but it turns out that complex numbers can be used to solve practical engineering problems and produce beautiful art.

Now let's look at zn+1 = zn2 + c but this time let's permit complex numbers for z0 and c. If you picture a ruler with the real numbers written on it then just as we can picture the real numbers arranged in a line we can also arrange the complex numbers geometrically but the result isn't a line but a plane. As well as moving left or right by increasing or decreasing the real number part we can also move up and down by increasing and decreasing how many multiples of the square root of minus one we have. So now we can take any position on the computer screen and describe its location using the complex number x + yi. For each pixel on the screen we use our formula zn+1 = zn2 + c to generate a sequence of complex numbers starting from z0 = x + yi. We use different values for x and y depending on the pixel but we use a fixed value for c. If the sequence of numbers grows larger and larger then we choose a colour for the pixel based on how quickly the numbers start growing big. If the sequence never produces big numbers, for example if the numbers shrink down towards 0 + 0i, then we give that pixel a fixed colour. Here I've left these pixels transparent. You can experiment with a variety of different formulas for calculating zn+1.
