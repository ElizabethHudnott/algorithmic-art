Creating Your Own Sketches

Writing The Code

This article discusses creating your own generative art sketches that take advantage of the environment I've created. The code template you will need to follow is shown below.

Each sketch has a title, which is used in GUI elements, and a hasRandomness property. If your sketch's output is deterministic and driven only by input parameters then this should be set to false. On the other hand if you use pseudorandom numbers then set it to true and the environment will add the build icon to the program's toolbar so users can generate a fresh image without adjusting the sketch's parameters. You can populate the helpFile property with a URL directing the environment towards a HTML file containing help information.

The optionsDocument property is technically optional and it's useful to omit it during an early development phase, but usually you will want to include it unless your intention is to deliberately create a static sketch without adjustable parameters. In another file, create an interface for adjusting the parameters using HTML and Bootstrap. Be sure to give your element IDs a short prefix in order to avoid name clashes with elements that are part of the environment itself. For example, in the Julia Set sketch I give each element an ID beginning with the prefix julia-.

The remainder of the code has three significant sections.

  1. The event handlers inside the callback to downloadFile(), which link the HTML code to the sketch's JavaScript parameters.
  2. The animatable property, which is a declarative description of how the sketch can be animated.
  3. The code that draws the sketch onto the screen, which is contained inside the generate() method.

There is a particularly challenging need to be able to redraw the sketch quickly in response to events like dragging a slider, while at the same time having the desire to create complex and intricate sketches that arouse artistic interest. There are two measures available which you should use to protect the application's performance.

  • The generate() method is generator function, which means you can use the yield statement periodically during time consuming drawing operations to temporarily suspend drawing and return control to the environment. The environment will then decide whether to continue with the present drawing operation, or alternatively if a parameter's value is changing too quickly then it might decide to abandon the current drawing and start again.
  • The generate() has a parameter called preview that accepts a numeric argument. A value of 0 always means that a complete redraw with the maximum level of detail has been requested. However, you are free to use other values inside your GUI's event handlers in any way that's useful to you. In some situations the environment will request preview level 1.

For instance, my phyllotaxis sketch uses four values for the preview parameter.

  1. A complete redraw, including recalculation of the petals' positions and their colours. The positions are cached to speed up subsequent calls to generate() when possible.
  2. Only a limited number of petals are drawn in order to give an impression of what the finished image would look like, but each petal does have all of its features calculated from scratch.
  3. All of the petals are drawn but none of them have their positions recalculated. The positions are retrieved from the cache. This is used when a parameter is changed which affects the petals' shapes or colours but not their locations.
  4. Only a limited number of petals are drawn and they don't have their positions recalculated. This is used when a visual effect is applied that is time consuming to render for a large number of petals even when much of numerical data is pre-calculated.

Alpha Transparency

Rather than setting the canvas' globalAlpha property to an absolute value, your sketch should instead multiply the property's existing value by the amount of alpha you want to set. This will ensure the sketch is composed properly against the background layer.

Random Numbers

To ensure animations run smoothly you should use the random.next() function instead of Math.random(). This ensures the nth random number requested by generate() when rendering the middle frame of the animation is calculated as being the average of the nth random number used in the start frame and the nth random number used in the end frame. The random.next() function has the additional advantage that the user can record the numbers used to seed this random number generator and use them to reproduce the same sequence of random numbers (and therefore exactly the same image) in a future creative session.

Intrinsic Time

If your constructor contains the line this.tween = 0; then the environment interprets this as meaning the sketch has an intrinsic sense of time. That is to say, it is capable of rendering more than one image from the same set of parameter values. The environment will set tween to a number between zero and one before it calls generate(), where zero represents the start frame and one represents the end frame. When a sketch uses the tween property a user can use animation features right away without the need to explicitly mark distinct start and end frames, although they can still do that if they wish.

Data Attributes

You can use a <button> with a data-reset attribute to reset a control and its corresponding JavaScript property to an initial value (which is determined by inspecting the value attribute). For example, <button type="button" data-reset="carpet-stretch"> will reset the <input> element with carpet-stretch as its ID and it will also reset the JavaScript property named stretch.

Writing the Help File

Each piece of context sensitive help information should be contained within a single HTML tag (though the tag can contain further HTML tags). Mark up tags in the help file using the id attribute to link help information with the appropriate context.

  • Use a tag with id="about" to mark content for inclusion in the About this Sketch box.
  • Other IDs in the help file will be matched with elements in the options document when they share the same ID. For example, if you create a control in the options document using <input id="mysketch-option1"… then you create help information for that option using <div id="mysketch-option1">.

Use <span class="sketch-title"> to format the title of the sketch.

Testing Your Sketch

Upload your files to a local web server that supports CORS. The Node module local-web-server can be used for this. Then use the following form to generate a URL.

http://localhost/
This should be the URL of your JavaScript file.

WebGL2 Fragment Shaders

As well as using the 2D canvas API you also have the option of expressing yourself artistically by writing code in the form of a WebGL fragment shader. Sketches that use shaders have much faster performance and create a better user experience than the 2D canvas for designs that are calculation intensive. Although hardware acceleration features of the 3D graphics card are used to speed up rendering in this section we're not discussing creating 3D shapes.

A sketch that uses a shader must contain the line this.isShader = true; in its constructor and the code for the shader must be contained in a file with the same name and location as the JavaScript file but with the .frag extension. For convenience canvasWidth, canvasHeight and tween are predefined as float variables and preview as int, PI as a constant and the hsla function (with four float parameters), all with the same meanings as for 2D canvas sketches. The out variable is named fragColor.

There are some differences compared to 2D canvas sketches. Only properties which are declared as animatable get passed to the shader. To ensure new values get passed to the shader, in most cases you won't want to assign to JavaScript properties directly inside your event handlers but instead you'll call setBgProperty(me, 'name', value);. A word about type conversion is needed. Properties declared in the continuous, pairedContinuous and xy sections are passed to the shader as float, vec2, vec3, vec4 and matrices and arrays of these types. Properties declared in the stepped and pairedStepped sections are passed as int, bool, ivec2, ivec3, ivec4, bvec2, bvec3 or bvec4, or as an array of one of these types. The necessary GSGL declarations will be written for you automatically. In your JavaScript code you can have a property that is an array of arrays and it will automatically be converted into a matrix or an array of vectors when it's passed to the shader. However, the length of an array typed property should not change after the sketch is initialized. When using an array of arrays the individual array components must all have the same length.

There are a couple of variant formats, which are particularly useful when working with arrays. The two parameter format setBgProperty(me, 'name'); forwards the current value of me.name to the shader, which is useful when you've altered some elements of an array property directly. This may be easier than passing a whole replacement array in one using the three parameter version. Finally, if you only need to update a single array element then there's setPropertyElement(me, 'name', index, value);
