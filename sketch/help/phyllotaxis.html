Phyllotaxis shows what happens when you keep adding one more item a little bit further down the line. The line in question spirals out from the centre of the image towards the sides but in a phyllotaxis simulation we don't usually draw the line itself. Instead we draw some figures (for example some circles) placed at fixed points spaced out along that invisible line. Curiously our minds tend to join up the dots to create the impression of a different spiral pattern that's distinct from the one the code is actually calculating! Many plants have their seeds, petals or leaves arranged in these spiral patterns. The conventional introductory example encourages us think about the seeds on the head of a sunflower. The formulas for calculating the position of the nth "petal" are, using polar coordinates:

  • r = c × nk
  • φ = nα

r is the straight line distance from the centre of the image to the nth petal. Each successive petal has a bigger value for r than the previous one, which is why the points are organized into a spiral rather than a circle.

φ is the angle between a straight line running horizontally through the centre of the image and the aforementioned line between the centre point and the petal.

α is a number that remains constant throughout the computation of the whole image. The default value is approximately 222.5°, so when the second petal is plotted we've already turned through more than 360°.

c is a constant scaling factor that just generally spaces the petals out more.

k is another constant and is conventionally set to 0.5, but we can generate a variety of tighter or looser spirals by substituting different values.
